## 问题二：app.py中调用不存在的main()函数

### 错误信息
```
NameError: name 'main' is not defined
File "requirement-agent\app.py", line 378, in <module>

```

### 问题原因
`app.py` 文件末尾调用了 `main()` 函数，但Streamlit应用不需要main函数。

### 解决方案
将 `if __name__ == "__main__": main()` 替换为注释 `# Streamlit runs this file directly, no main() needed`

## 问题三：界面纯英文，需要支持中英文切换

### 需求
界面默认显示中文，支持中英文切换。

### 解决方案
1. 添加 `TEXTS` 中英文文本字典
2. 添加 `t()` 翻译函数
3. 侧边栏添加语言选择器（中文/English）
4. 将所有界面文本改为通过 `t()` 函数获取

### 关键代码
```python
TEXTS = {
    'en': { 'page_title': 'Requirement Flow...', ... },
    'zh': { 'page_title': '需求流程...', ... }
}

def t(key, *args):
    lang = st.session_state.get('lang', 'zh')
    text = TEXTS[lang].get(key, key)
    return text.format(*args) if args else text
```

### 修改文件
- `app.py` (大量修改，添加i18n支持)

## 问题七：对话界面显示JSON内容

### 问题描述
用户在对话界面看到 `[REQUIREMENT_CARD]` 和JSON内容，体验不好。

### 问题原因
1. AI回复中包含 `[REQUIREMENT_CARD]` 和JSON内容
2. 代码直接将完整回复（含JSON）显示给用户

### 解决方案
1. 修改 `clarify_system.txt` 提示词，明确要求AI：
   - 信息不足时只输出对话（不输出JSON）
   - 信息充足时只输出 `[REQUIREMENT_CARD]` 和JSON（不输出额外文本）
2. 修改 `clarify_agent.py`，当检测到需求卡片标记时：
   - 只返回友好的提示信息
   - 不显示JSON内容给用户

### 修改文件
- `prompts/clarify_system.txt`
- `agents/clarify_agent.py`

## 问题九：orchestrator变量作用域错误

### 错误信息
```
生成失败: name 'orchestrator' is not defined
```

### 问题原因
`orchestrator` 变量在 `run_generation()` 函数内部定义：
```python
async def run_generation():
    orchestrator = st.session_state.orchestrator
    ...
```

但在函数外部（第390行）尝试访问：
```python
st.session_state.session = orchestrator.session  # orchestrator 已不在作用域内
```

### 解决方案
将 `orchestrator.session` 改为 `st.session_state.orchestrator.session`

### 修改文件
- `app.py`
