# 需求流程自动化智能体系统 - 使用场景

## 场景一：产品经理快速出PRD

**用户**：产品经理小王  
**背景**：老板说"做个用户积分兑换功能"，需要快速产出完整PRD

**操作**：
1. 打开 http://localhost:8501
2. 在对话框输入："我想做一个用户积分兑换商品的功能"
3. 智能体自动追问澄清（用户角色、积分规则、兑换流程等）
4. 澄清完成后自动生成：
   - 完整PRD文档
   - 技术方案（API设计、数据库变更）
   - 测试用例（P0/P1/P2三个优先级）
   - 风险评估报告

**价值**：从一句话需求到完整交付文档，原本需要2-3天的工作，30分钟内完成

---

## 场景二：技术负责人评估工作量

**用户**：技术Leader小李  
**背景**：拿到PRD后需要快速评估前后端和测试的工作量

**操作**：
1. 输入需求："开发一个社区团购的团长管理后台"
2. 经过3-5轮澄清对话
3. 查看生成的**技术方案**标签页：
   - 涉及模块列表
   - 新增/修改的API清单
   - 数据库变更说明
   - **工作量估算**：前端5天、后端8天、测试3天

**价值**：快速获得工作量评估，辅助排期决策

---

## 场景三：测试人员获取测试用例

**用户**：测试工程师小张  
**背景**：新功能上线前需要完整测试用例覆盖

**操作**：
1. 输入需求或通过历史记录加载已有需求
2. 系统自动生成三类用例：
   - **P0主流程**：5+个核心流程用例
   - **P1异常场景**：3+个异常处理用例
   - **P2边界场景**：3+个边界条件用例
3. 点击"✅ Tests JSON"下载用例文件

**价值**：不再遗漏异常场景和边界条件，提升测试覆盖率

---

## 场景四：项目经理风险评估

**用户**：项目经理小赵  
**背景**：新功能上线前需要评估技术风险和业务风险

**操作**：
1. 加载已完成的需求会话
2. 查看**风险评估**标签页：
   - 风险等级（高/中/低）
   - 具体风险点列表
   - 改进建议
   - 未覆盖场景提示
   - 是否需要人工审核

**价值**：提前识别风险点，制定应对策略

---

## 场景五：新人接手项目

**用户**：刚入职的开发小陈  
**背景**：接手一个已有项目，需要快速理解需求和技术方案

**操作**：
1. 侧边栏点击历史会话
2. 查看完整的需求卡片、PRD、技术方案
3. 通过下载按钮获取JSON格式文档
4. 中英文切换方便外籍同事协作

**价值**：快速了解项目全貌，降低沟通成本

## 快速开始

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑 .env 填入 OPENAI_API_KEY

# 2. 安装依赖（如未安装）
venv/Scripts/pip install -r requirements.txt

# 3. 启动应用
venv/Scripts/streamlit run app.py

# 4. 访问
http://localhost:8501
```
