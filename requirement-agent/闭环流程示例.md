# 闭环流程示例：从一句话需求到完整交付文档

以**场景一：产品经理快速出PRD**为例，给出完整闭环流程。

---

## 闭环流程：从一句话需求到完整交付文档

### 第一步：启动与初始化

**用户操作**：
```bash
# 启动系统
cd E:/xuan/code/Autoflow/requirement-agent
venv/Scripts/streamlit run app.py
```

**系统行为**：
- 自动打开浏览器 http://localhost:8501
- 显示中文界面（默认）
- 侧边栏显示"无活动会话"
- 主界面显示输入框："输入您的需求（一句话即可）"

---

### 第二步：输入需求

**用户操作**：
在输入框输入："我想做一个用户积分兑换商品的功能"

**系统行为**：
1. 消息显示：
   ```
   👤 用户：我想做一个用户积分兑换商品的功能
   🤖 智能体思考中...
   ```
2. 调用 `ClarifyAgent` 进行多轮澄清
3. 更新进度条：10% - "需求澄清中... (第1/5轮)"

---

### 第三步：多轮澄清对话

**第1轮 - 智能体追问**：
```
🤖 为了完善需求卡片，我需要了解以下信息：
1. 用户角色有哪些？（例如：普通用户、VIP用户、管理员）
2. 积分如何获取？（例如：购物获得、签到获得、活动奖励）
3. 兑换比例是多少？（例如：100积分=1元）
4. 是否有积分有效期？
```

**用户回复**：
"用户角色有普通用户和VIP用户。积分通过购物获得，每消费1元得1积分。100积分=1元。积分有效期1年。"

**第2轮 - 智能体继续追问**：
```
🤖 还需要确认：
1. 可兑换的商品类型有哪些限制？
2. 兑换流程是怎样的？（例如：选择商品→确认兑换→扣减积分→生成订单）
3. 是否支持部分积分+部分现金支付？
4. 兑换后是否支持退款？退款时积分如何返还？
```

**用户回复**：
"可兑换实物商品和虚拟商品（优惠券）。流程是：选商品→确认→扣积分→生成订单。不支持混合支付。兑换后15天内可退款，退款时积分原路返还。"

**第3轮 - 智能体生成需求卡片**：
```
🤖 信息已收集完整，需求卡片已生成：

名称：用户积分兑换商品功能
背景：为提升用户粘性，建立积分体系让用户通过消费累积积分并兑换商品
用户角色：普通用户、VIP用户
核心操作：查看积分余额、浏览可兑换商品、兑换商品、查看兑换记录
约束条件：不支持混合支付、积分有效期1年
范围外：积分转账、积分赠送功能
技术栈：常见Web技术栈

是否确认？如有补充请继续输入。
```

**用户回复**："确认，开始生成文档"

---

### 第四步：自动生成全套文档

**系统自动执行**（用户无需操作）：

```
进度条变化：
20% - 澄清完成！开始生成文档...
30% - 开始生成PRD...
40% - PRD生成完成！
50% - 并行运行技术方案和测试用例智能体...
70% - 技术方案和测试用例生成完成！
90% - 风险评估完成！
100% - 所有任务完成！
```

**后台并行执行**：
1. `PRDAgent` - 生成产品需求文档
2. `TechAgent` - 生成技术方案（并行）
3. `TestAgent` - 生成测试用例（并行）
4. `RiskAgent` - 生成风险评估报告

#### ⚠️ 风险评估
```
风险等级：🟡 Medium

风险点：
- 积分扣减并发问题，可能导致超扣
- 兑换后退款可能带来资金损失
- 未做防刷机制，可能被恶意刷积分

改进建议：
- 使用Redis分布式锁保证积分扣减原子性
- 退款时增加人工审核流程
- 增加风控规则：单日兑换上限、新注册用户限制

未覆盖场景：
- 系统宕机时积分已扣但订单未生成
- 商品下架后用户已兑换但未发货

是否需要人工审核：☑️ 是
```

## 闭环总结

```
输入：一句话需求
  ↓
[多轮澄清对话] → 需求卡片
  ↓
[并行生成] → PRD + 技术方案 + 测试用例
  ↓
[风险评估] → 风险报告
  ↓
输出：完整文档包（Markdown + JSON）
  ↓
[可选] 历史加载 → 迭代更新
```

**总耗时**：澄清5分钟 + 生成10分钟 = **15分钟**完成原本2-3天的工作

---

## 关键智能体说明

| 智能体 | 功能 | 输出 |
|--------|------|------|
| ClarifyAgent | 多轮对话澄清需求 | 需求卡片 (RequirementCard) |
| PRDAgent | 生成产品需求文档 | PRD文档 (PRDDocument) |
| TechAgent | 生成技术方案 | 技术方案 (TechPlan) |
| TestAgent | 生成测试用例 | 测试用例 (TestCases) |
| RiskAgent | 评估项目风险 | 风险报告 (RiskReport) |

---

## 系统架构

```
用户输入
   ↓
[ClarifyAgent] ← 多轮澄清 → 需求卡片
   ↓
   ├→ [PRDAgent] → PRD文档
   ├→ [TechAgent] → 技术方案 (并行)
   └→ [TestAgent] → 测试用例 (并行)
   ↓
[RiskAgent] → 风险报告
   ↓
输出所有文档 + 保存到历史
```

---

## 文件清单

生成的完整文档包包含：

1. **full_report.md** - 人类可读的完整报告（Markdown格式）
2. **requirement_card.json** - 结构化需求卡片
3. **prd.json** - 结构化PRD文档
4. **tech_plan.json** - 结构化技术方案
5. **test_cases.json** - 结构化测试用例
6. **risk_report.json** - 结构化风险评估

所有JSON文件均符合对应的Pydantic模型定义，可直接用于后续自动化流程。
